# The Nirvana Architecture 🧠

**A self-evolving AI engineering constitution.** Drop one file into your workspace and your AI
coding agent joins a global network that gets smarter every two weeks — powered by real discoveries
from every developer using it.

## How to join

1. **Fork this repository.** Your fork is your node on the network.
2. Copy `NIRVANA.md` into your project root as `CLAUDE.md`, `GEMINI.md`, `AGENTS.md`, or
   `.cursorrules` (whichever your AI tool reads). Copy `EVOLUTIONS.md` alongside it.
3. Start working. Your AI reads the Kernel, syncs the latest global patterns from this repo's
   `EVOLUTIONS.md`, and — with your consent — logs its own anonymized breakthroughs to your fork.

## How it evolves

```
your fork ──┐
other forks ─┼──▶ 🤖 synthesis bot (every 14 days) ──▶ Pull Request ──▶ human review ──▶ main
curated sources ┘                                                                        │
                    every fork syncs the upgraded EVOLUTIONS.md on its next session ◀────┘
```

- `NIRVANA.md` is the **Kernel** — the immutable constitution. No automated process ever rewrites it.
- `EVOLUTIONS.md` is the **living brain** — a pattern library synthesized from fork discoveries and
  curated sources (Karpathy's context engineering, 12-factor agents, and more).
- Every evolution is reviewed by a human before it ships. Harvested fork content is treated as
  untrusted data with strict filters: no injected instructions, no secrets, no poison patterns ever
  make it into the core.

## What your AI learns to do

- **Context engineering** — one living `ARCHITECTURE.md` as the session's source of truth
- **Self-annealing** — every fixed bug leaves a permanent lesson in the workspace
- **The determinism boundary** — LLMs decide, deterministic scripts execute
- **Legacy protection** — finalized code is tagged and never silently rewritten
- **Safe evolution** — new features built beside old ones behind flags, never over them

## Privacy

The Hive Mind shares **engineering patterns only** — abstract lessons with all names, paths, keys,
URLs, and business logic stripped. Telemetry is consent-based and can be disabled with
`NIRVANA_TELEMETRY=off`. Read the full notice at the top of `NIRVANA.md`.
